# Автоматизация CHANGELOG

## Как это работает

Интерактивный скрипт запускается вручную перед каждым PR.  
Он показывает последние коммиты как контекст и помогает сформировать запись в `CHANGELOG.md`.

## Использование

```bash
npm run changelog
```

## Структура файлов

```
scripts/
  changelog.mjs  — интерактивный скрипт обновления CHANGELOG
```

## Автоматизация через git-хук (опционально)

Чтобы скрипт запускался автоматически перед каждым `git push`, создайте файл `.git/hooks/pre-push`:

```sh
#!/bin/sh
node scripts/changelog.mjs
```

На macOS / Linux выдайте права на выполнение:
```bash
chmod +x .git/hooks/pre-push
```

## Формат записи в CHANGELOG.md

```md
## [6.80.3] - 08.07.2026

### Исправлено
- функциональность "Форма" / "Чекбокс"
    - исправлено состояние чекбокса при сабмите
    - убрана лишняя валидация
```

## Категории

| Выбор | Заголовок     | Когда использовать                        |
|-------|---------------|-------------------------------------------|
| 1     | Добавлено     | Новая функциональность                    |
| 2     | Изменено      | Изменение существующей функциональности   |
| 3     | Исправлено    | Исправление бага                          |
| 4     | Удалено       | Удалённая функциональность                |
| 5     | Общее         | Инфраструктура, зависимости, документация |

## Версия

| Выбор | Пример          | Когда использовать              |
|-------|-----------------|----------------------------------|
| 1     | 6.80.2 → 6.80.3 | Баг-фикс, мелкие правки          |
| 2     | 6.80.2 → 6.81.0 | Новая функциональность           |
| 3     | 6.80.2 → 7.0.0  | Ломающие изменения               |
| 4     | без изменений   | Правки документации, рефакторинг |
