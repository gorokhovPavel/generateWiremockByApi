# Обновление CHANGELOG

Раньше запись в `CHANGELOG.md` вносилась вручную перед каждым PR.
Теперь это делает скрипт — он задаёт вопросы и сам записывает всё в нужном формате.

## Для разработчика: что изменилось

**Было:** открыл `CHANGELOG.md`, вручную написал версию, дату, категорию, описание.

**Стало:** запустил команду, ответил на вопросы, скрипт сделал всё сам — включая коммит.

---

## Как пользоваться

Перед тем как создать PR, запусти в корне проекта:

```bash
npm run changelog
```

Скрипт покажет последние коммиты как подсказку и проведёт по шагам:

```
Последние коммиты:
  a1b2c3d fix checkbox state on submit
  d4e5f6g add validation to form fields

Обновить CHANGELOG? (y/n): y

Текущая версия: 6.80.2
Выберите тип обновления версии package.json:
  1) patch        ← баг-фикс, мелкие правки
  2) minor        ← новая функциональность
  3) major        ← ломающие изменения
  4) не обновляем ← правки документации, рефакторинг
Выбор [1]: 1
Новая версия: 6.80.3

Категория:
  1) Добавлено
  2) Изменено
  3) Исправлено
  4) Удалено
  5) Общее
Выбор [1]: 3

Функциональность (Enter — пропустить): Форма
Подраздел (Enter — пропустить): Чекбокс
Описание: исправлено состояние чекбокса при сабмите
Ещё одно описание? (Enter — пропустить): убрана лишняя валидация
Ещё одно описание? (Enter — пропустить): ← Enter

Сделать коммит сразу? (y/n) [y]: y

✓ CHANGELOG.md, package.json, package-lock.json обновлены и закоммичены → версия 6.80.3
```

Результат в `CHANGELOG.md`:

```md
## [6.80.3] - 08.07.2026

### Исправлено
- функциональность "Форма" / "Чекбокс"
    - исправлено состояние чекбокса при сабмите
    - убрана лишняя валидация
```

---

## Подсказки

**Функциональность и подраздел** — необязательны. Если изменение не привязано к конкретному экрану или разделу, нажми Enter и перейди сразу к описанию.

**Несколько строк описания** — вводи по одной, каждый раз нажимая Enter. Когда закончил — просто нажми Enter на пустой строке.

**Коммит сразу** — если ответить `n`, скрипт обновит файлы, но не закоммитит. Можно добавить их в свой коммит вручную:
```bash
git add CHANGELOG.md package.json package-lock.json
```

**Версия не изменяется** — выбери пункт `4) не обновляем`. `package.json` и `package-lock.json` трогаться не будут.

## Справочник категорий

| Категория  | Когда использовать                        |
|------------|-------------------------------------------|
| Добавлено  | Новая функциональность                    |
| Изменено   | Изменение существующей функциональности   |
| Исправлено | Исправление бага                          |
| Удалено    | Удалённая функциональность                |
| Общее      | Инфраструктура, зависимости, документация |

## Автоматический запуск при git push (опционально)

Создай файл `.git/hooks/pre-push`:

```sh
#!/bin/sh
node scripts/changelog.mjs
```

На macOS / Linux:
```bash
chmod +x .git/hooks/pre-push
```

После этого скрипт будет запускаться автоматически перед каждым `git push`. Если обновлять лог не нужно — ответь `n`, пуш продолжится.
