# Автоматизация CHANGELOG

## Как это работает

Перед каждым `git push` автоматически запускается интерактивный скрипт.  
Он показывает последние коммиты как контекст и предлагает добавить запись в `CHANGELOG.md`.  
Если запись не нужна — просто ответить `n`, пуш продолжится.

## Установка (один раз на проект)

```bash
npm run changelog:install
```

Команда создаёт `.git/hooks/pre-push` и выставляет права на выполнение.  
Нужно запустить каждому разработчику после клонирования репозитория.

## Использование

### Автоматически — при каждом push

```bash
git push origin feature/my-branch
```

```
Последние коммиты с момента прошлого changelog:
  abc1234 fix button disabled state
  def5678 add checkbox validation

Обновить CHANGELOG? (y/n): y

Bump type (patch/minor/major) [patch]: ↵
Новая версия: 6.80.3

Категория:
  1) Добавлено
  2) Изменено
  3) Исправлено
  4) Удалено
Выбор: 3

Описание: функциональность "Форма" — исправлено состояние чекбокса

✓ CHANGELOG.md обновлён и закоммичен
```

### Вручную — без push

```bash
npm run changelog
```

## Структура файлов

```
scripts/
  changelog.mjs        — интерактивный скрипт обновления CHANGELOG
  install-hooks.mjs    — установка git-хука pre-push
```

## Формат записи в CHANGELOG.md

```md
## [6.80.3] - 08.07.2026

### Исправлено
- функциональность "Форма" — исправлено состояние чекбокса
```

## Категории

| Выбор | Заголовок     | Когда использовать                        |
|-------|---------------|-------------------------------------------|
| 1     | Добавлено     | Новая функциональность                    |
| 2     | Изменено      | Изменение существующей функциональности   |
| 3     | Исправлено    | Исправление бага                          |
| 4     | Удалено       | Удалённая функциональность                |

## Bump type

| Тип   | Пример              | Когда использовать              |
|-------|---------------------|---------------------------------|
| patch | 6.80.2 → 6.80.3     | Баг-фикс, мелкие правки         |
| minor | 6.80.2 → 6.81.0     | Новая функциональность          |
| major | 6.80.2 → 7.0.0      | Ломающие изменения              |
